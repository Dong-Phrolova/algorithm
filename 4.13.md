## 线段树一些高级用法

+ 摩尔投票，出现次数严格大于一半的众数，最后需要再次扫描一次数组，确定唯一可能候选的$cnt$是否真的大于$n/2$。具有**区间可加性**，可以用线段树维护。**扩展**：出现次数大于$n/k$的数，需要开一个表记录$(k-1)$个可能的候选，扫描一遍数组，进行摩尔投票，只不过此时需要改为扫到第$k$种数，需要使得此时表中所有数$cnt-1$;

## 可持久化线段树（主席树）

+ 单点修改，记录每个版本的头节点编号，沿途所有节点新建，其余复用，$O(logn)$。
+ 单点修改，区间查询，**区间第k小数：权值线段树**：
  + 类似权值树状数组，维护$ cnt[i] $表示$i$这个数字出现次数
  + 我想得到$ [l,r] $范围上的$cnt$数组,只需得到前缀的$cnt$数组
  + 因此，（先离散化），按顺序扫描一遍数组，维护每个前缀产生的$cnt$的历史版本。走到$i$的时候，生成第$i$个版本的线段树，表示$a[1]—a[i]范围内，每个数出现的次数$ ，每个节点维护$[l,r]$范围内有多少数字即可
+ 感性理解为**“前缀线段树”**，差分后得到想要的信息
  + 类似的可以推广到**树上第k小**，只需要在树上差分，P2680
  
+ 范围修改，范围查询，引入lazy tag，但是查询时也需要下发，新建一组左右儿子，因此复杂度是$O(4n+nlogn+mlogn)$
+ 标记永久化
+ 一些神秘技巧：
  + 维护某数第一次出现的位置，维护区间这种位置有多少个，**倒序建立可持久化线段树**
  + **在线**查询区间$mex$，见下方

## 学习一下区间mex

给定一个集合$s$

+ 要求支持1.插入一个数2.查询s的mex
  + 只需维护vis数组，表示i是否出现过。指针p从0开始，如果此时的位置改为1，则暴力向后移动，直到下一个0，即为所求。注意到p只会增加，复杂度$O(n)$;

+ 要求增加删除功能，只需维护一个set，表示未出现过的数，初始插入0~n，每次插入时，删除这个数，删除时如果没有就加入，查询取最小值即可，$O(nlogn)$
+ **分块**：将0~n分成$\sqrt n$块，cnt维护i出现了多少次，sum表示第i块出现了多少个数，此时插入和删除是O(1)的，查询时，**找到第一个没有放满的块**，然后在块内暴力找，复杂度是$O(n*\sqrt n)$

给定数组a

+ 离线+线段树1：
  + $mex[i]$表示$[1,i]$的mex，ne[i]表示a[i]下次出现的位置。先扫描一遍数组，O(n)求出mex和计算ne数组。
  + ne数组计算：开一个last数组，表示i上次出现的位置，**倒序扫描**更新ne和last即可。
  + 我们考虑移动左端点，即删除a[i]，会影响哪些mex。
  + 右端点在[ne[i],n]的区间mex不受影响，在[i,ne[i]-1]的区间，会少一个a[i]，那些mex比a[i]大的均会更新为a[i]，是**吉如一线段树**能做到的。
  + 注意到，**mex数组具有单调性**，因此，我们可以在[i,ne[i]-1]二分mex第一个大于a[i]的位置pos，然后[i,pos-1]区间重置为a[i]即可。
  + 移动一次左端点，二分复杂度为O(logn)，区间重置复杂度O(logn)，左端点共移动n次，总复杂度O(nlogn)
  + 增加sum信息，可维护**区间mex和**
  + 离线处理询问，用左端点排序，每次回答以l为左端点的全部询问。（不能在线因为每次移动左端点无法恢复）

+ 离线+权值线段树

  + 叶节点储存**i在数组中最后一次出现的位置**，非叶节点储存**区间min**
  + 移动右端点，每次扫描一a[i]，找到a[i]进行如下操作：
    + 更新储存a[i]的位置的叶节点为i
    + 处理右端点为i的询问：对于询问[l,i]，**找一个数v，last[v]<l，而且是最小的**，这就是说，v是mex候选，最小就是mex定义

  + 查询时，如果左儿子min<l，说明较小的数值范围内有一个数上次出现的位置不在查询区间内，进入左子树，否则说明左侧这些较小的数都在[l,r]出现过，则进入右子树。
  + 仍然是对查询排序，一次处理右端点相同的所有询问

+ 在线+可持久化权值线段树
  + 只需要在每次移动右端点的时候，进行主席树的修改操作，对于询问[l,r]只需要在root[r]这棵树上进行即可
